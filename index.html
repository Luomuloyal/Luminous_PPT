04-A. 客户端与服务端协同

📱 Flutter App 层

• Riverpod 管理全局状态与依赖注入
• 本地缓存 + 离线读取，提升弱网可用性
• 页面组件拆分，支持快速迭代与复用
• 多主题视觉系统，兼顾品牌一致性

⚙️ API 服务层

• TypeScript 提供端到端类型约束
• 分层架构拆分 Controller/Service/DAO
• Redis 热点缓存降低数据库压力
• 日志追踪 + 错误监控保障稳定性

04-B. 数据与安全治理

🗄️ 数据分层设计

• MongoDB：健康记录、问诊上下文、行为日志
• MySQL：药品信息、规则表、运营配置
• 统一数据访问层，提升查询可维护性
• 增量备份 + 异地副本保障业务连续性

🔐 安全合规策略

• JWT 双 Token + 刷新策略控制会话生命周期
• 传输层 TLS 加密 + 关键字段脱敏存储
• 大模型访问鉴权、频控与审计留痕
• 异常流量拦截与熔断，防止接口雪崩

04-C. 工程化与部署能力

🚀 CI/CD

• 自动化构建与测试
• 多环境配置隔离

📈 Observability

🤝 Team Workflow
